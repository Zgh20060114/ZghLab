- **在一个部分可观测的环境里，智能体是如何通过内部模型来理解世界、做出决策、并更新自己认知的。**

## 先分清两套系统

| 符号 | 全称 | 含义 |
|------|------|------|
| \(w_t\) | 环境真实状态 | 真实世界的隐藏状态，智能体看不到 |
| \(z_t\) | 智能体内部状态 | 智能体对世界的“信念”或“记忆” |
| \(a_t\) | 动作 | 智能体输出的行为 |
| \(o_t\) | 观测 | 智能体从环境收到的感知信号（如摄像头图像） |

---

## 一个时间步 t 发生了什么？

### 1. 智能体做决策
- 当前智能体的内部信念是 \(z_t\)
- 根据策略 \(\pi_t\) 选择动作：  
  $a_t \sim \pi_t(z_t \mid \theta_t)$
  意思是：在给定内部状态 \(z_t\) 和策略参数 \(\theta_t\) 的情况下，动作 \(a_t\) 是从这个策略中采样得到的。

---

### 2. 环境更新（真实世界）
- 环境有一个隐藏的真实状态 \(w_t\)
- 环境根据环境模型 \(M\) 更新状态：
  $w_{t+1} \sim M(w_t, a_t)$
  意思是：真实世界会转移到下一个状态 \(w_{t+1}\)，这个转移是随机的。

---

### 3. 环境发出观测
- 根据环境的观测模型 \(O\)，产生观测：
  $o_{t+1} \sim O(w_{t+1})$
  意思是：真实世界的新状态会产生一个观测信号 \(o_{t+1}\)，比如一张图片、一个传感器读数。

---

### 4. 智能体处理观测
- 智能体有一个观测编码器 \(E\)，(降维)把原始观测转换成向量：
  $e_{t+1} = E(o_{t+1})$
- 然后用状态更新函数 \(U\) 更新内部状态：
  $z_{t+1} = U(z_t, a_t, e_{t+1})$
- 如果使用可选的智能体预测,那么定义一个SU函数:状态更新函数U和预测函数P的复合:
  $z_{t+1} = SU(z_t, a_t, e_{t+1})$  
  等价于:
  $z_{t+1} = U(P(z_t, a_t), e_{t+1})$
  意思是：根据**旧信念 + 刚做的动作 + 新观测的编码**，更新自己对世界的理解。

---

### 5. 智能体预测（可选）
- 智能体还可以用预测函数 \(P\) 预测未来的内部状态：
  $\hat{z}_{t+1} = P(z_t, a_t)$
- 以及用观测解码器 \(D\) 预测会看到什么(反过来,可以作为一个有用的训练信号)：
  $\hat{o}_{t+1} = D(\hat{z}_{t+1})$

这些是“预测”部分，用于帮助智能体学习更好的世界模型。

---

### 6. 学习更新（慢时间尺度）
- 策略 \(\pi_t\) 由参数 \(\theta_t\) 决定
- 强化学习算法 \(A\) 会根据经验**慢慢地**更新 \(\theta_t\)

你问到了 POMDP 形式化的核心。这两个公式是用数学语言精确描述“世界的随机性从何而来”。

简单直接的回答是：**公式 1.14 和 1.15 是 POMDP 中两个核心随机过程的“生成式”定义。它们的作用是明确地告诉我们，环境和观测的随机性是由一些看不见的“随机种子” \( \epsilon \) 引入的。**

这两个公式是**等价**于我们之前讨论的随机转移函数 \(M\) 和随机观测函数 \(O\) 的，但提供了一种不同的、更具构造性的视角。

## 环境模型M和观测模型O的公式定义
## 一、公式 1.14：随机转移函数的生成式定义

$p(w_{t+1} | w_t, a_t) = \mathbb{E}_{\epsilon_t^w} \left[ \mathbb{I}(w_{t+1} = W(w_t, a_t, \epsilon_t^w)) \right]$

### 1. 它是什么？作用是什么？
-   **是什么**：这是对 **状态转移概率** \( p(w_{t+1} | w_t, a_t) \) 的另一种定义方式。
-   **作用**：它揭示了这种随机性是如何产生的。它告诉我们，有一个**确定性的函数** \(W\)，但函数的输入除了当前状态 \(w_t\) 和动作 \(a_t\) 之外，还有一个随机变量 \(\epsilon_t^w\)。所有的随机性都来源于这个 \(\epsilon_t^w\)。

### 2. 公式内容与含义拆解

-   **\(W(w_t, a_t, \epsilon_t^w)\)**：这是一个**确定性的函数**。给定 \(w_t\)、\(a_t\) 和一个具体的随机数 \(\epsilon_t^w\)，它能**唯一确定**地计算出下一个状态 \(w_{t+1}\)。
-   **\(\epsilon_t^w\)**：这是一个**随机变量**（例如，服从标准正态分布或均匀分布）。它是“随机性的源头”。
-   **\(\mathbb{I}(\text{condition})\)**：这是**指示函数**。如果括号里的条件为真，则函数值为 1；否则为 0。它在这里扮演了“计数器”的角色。
-   **\(\mathbb{E}_{\epsilon_t^w}[\cdot]\)**：这是对随机变量 \(\epsilon_t^w\) 求**期望**。

### 3. 整个公式的含义

这个公式实际上是在做一件事：**对所有可能的随机源 \(\epsilon_t^w\) 取平均，来计算“状态变为 \(w_{t+1}\)”这件事的概率**。

过程是这样的：
1.  我们有一个能产生随机数的黑箱子 \(\epsilon_t^w\)。
2.  对于每一个具体的随机数 \(\bar{\epsilon}\)，我们把它代入确定性函数 \(W\)，得到一个确定的下一个状态 \(W(w_t, a_t, \bar{\epsilon})\)。
3.  我们检查这个计算出的结果是否等于我们关心的那个特定状态 \(w_{t+1}\)。如果是，\( \mathbb{I} = 1\)；如果不是，\( \mathbb{I} = 0\)。
4.  最后，我们对所有可能的 \(\bar{\epsilon}\) 求期望（即加权平均）。这个平均值，就是我们从 \((w_t, a_t)\) 转移到 \(w_{t+1}\) 的真实概率 \(p(w_{t+1} | w_t, a_t)\)。

### 4. 一个简单例子

假设一个非常简单的环境：
-   \(W(w_t, a_t, \epsilon_t^w) = w_t + a_t + \epsilon_t^w\)，其中 \(\epsilon_t^w\) 是一个公平的硬币，正面（\(\epsilon=1\)）概率 0.5，反面（\(\epsilon=-1\)）概率 0.5。
-   当前 \(w_t = 0\)，\(a_t = 1\)。

我们想计算 \(p(w_{t+1}=2 | w_t=0, a_t=1)\)：

1.  当硬币为正面 (\(\epsilon=1\))，\(W = 0+1+1 = 2\)，符合条件，\(\mathbb{I}=1\)。
2.  当硬币为反面 (\(\epsilon=-1\))，\(W = 0+1-1 = 0\)，不符合条件，\(\mathbb{I}=0\)。
3.  求期望：\(0.5 \times 1 + 0.5 \times 0 = 0.5\)。

所以，\(p(w_{t+1}=2 | w_t=0, a_t=1) = 0.5\)。这与我们直观理解的随机转移完全一致。

## 二、公式 1.15：随机观测函数的生成式定义

$p(o_{t+1} | w_{t+1}) = \mathbb{E}_{\epsilon_{t+1}^o} \left[ \mathbb{I}(o_{t+1} = O(w_{t+1}, \epsilon_{t+1}^o)) \right] $

### 1. 它是什么？作用是什么？
-   **是什么**：这是对 **观测概率** \( p(o_{t+1} | w_{t+1}) \) 的生成式定义。
-   **作用**：它与公式 1.14 的原理完全一样，只是对象换成了观测。它表明，观测的随机性也是由一个**确定性的函数** \(O\) 和一个独立的**随机源** \(\epsilon^o\) 共同产生的。

### 2. 公式内容与含义拆解

-   **\(O(w_{t+1}, \epsilon_{t+1}^o)\)**：确定性的观测生成函数。给定真实状态 \(w_{t+1}\) 和一个具体的随机数 \(\epsilon_{t+1}^o\)，它唯一确定地计算出智能体将收到的观测 \(o_{t+1}\)。
-   **\(\epsilon_{t+1}^o\)**：另一个独立的随机变量，是观测随机性的源头。
-   **\(\mathbb{I}\) 和 \(\mathbb{E}\)**：与公式 1.14 中的角色完全一样。

整个公式的含义就是：对所有可能的观测随机源 \(\epsilon_{t+1}^o\) 取平均，来计算“在真实状态 \(w_{t+1}\) 下获得观测 \(o_{t+1}\)”的概率。

---

## 三、总结：为什么要写成这种形式？

1.  **理论分析的便利**：这种“确定性函数 + 随机源”的表述，在数学上被称为**重参数化技巧**。它把随机性从函数内部剥离出来，使得在分析一些性质（如可微性）时更加方便。
2.  **计算机模拟的基础**：这正是我们在计算机程序中模拟随机环境的方式。我们不会真的去存储一个无穷大的概率表，而是定义一个确定性函数（如 `next_state = W(state, action, noise)`），然后每次调用时，都从一个随机数生成器（即 \(\epsilon\) 的采样器）中传入一个不同的 `noise`。这样就能生成符合概率分布的随机轨迹。
3.  **连接理论与实现**：它完美地连接了抽象的数学定义（概率分布）和具体的计算机实现（确定性函数+随机种子）。

所以，这两个公式不是要取代我们之前对 \(M\) 和 \(O\) 的理解，而是为我们提供了一个更底层、更具操作性的视角，来看待 POMDP 中的随机性是如何一步一步产生的。

## 一、从贝尔曼方程开始

公式 (1.31) 是贝尔曼最优方程（状态价值形式）：
\[
V^*(s) = \max_a \left[ R(s,a) + \gamma \sum_{s'} p(s' \mid s,a) V^*(s') \right]
\]

对于**固定策略** \(\pi\)（不是最优），贝尔曼方程是：
\[
V^\pi(s) = R(s,a) + \gamma \sum_{s'} p(s' \mid s,a) V^\pi(s')
\]
其中 \(a = \pi(s)\)（确定性策略）或对 \(a\) 取期望。

---

## 二、两个核心近似

### 1. 采样近似期望
贝尔曼方程里有期望 \(\sum_{s'} p(s'|s,a) V^\pi(s')\)，但我们不知道 \(p\)，也无法枚举所有 \(s'\)。

用**采样**代替：
- 从真实环境中采样一个 \(s' \sim p(\cdot|s,a)\)
- 用 \(r + \gamma V^\pi(s')\) 作为对 \(R(s,a) + \gamma \sum p V^\pi(s')\) 的**无偏估计**

这就是**蒙特卡洛思想**：用单个样本估计期望。

---

### 2. 增量更新（梯度下降思想）
我们想让 \(V(s)\) 接近目标值 \(r + \gamma V(s')\)。

定义误差：
\[
\delta = r + \gamma V(s') - V(s)
\]

用**梯度下降**更新 \(V(s)\)：
- 目标函数：\(\frac{1}{2} \delta^2\)
- 梯度：\(-\delta\)
- 更新：\(V(s) \leftarrow V(s) + \eta \delta\)

这就是公式 (1.32)：
\[
V(s) \leftarrow V(s) + \eta [r + \gamma V(s') - V(s)]
\]

---

## 三、为什么叫 Temporal Difference (TD)？

因为 \(\delta = r + \gamma V(s') - V(s)\) 是**不同时间点的价值估计之差**：
- \(V(s)\)：当前状态的价值估计（当前时间）
- \(r + \gamma V(s')\)：用下一步奖励和价值估计的未来（未来时间）

这个差值就是 **TD 误差**，用来调整当前估计。


一句话：**TD 学习是用采样 + 梯度下降来近似求解贝尔曼方程，每次用单步采样更新价值估计。**

**另一种建模部分可观测系统的方法——预测状态表示（PSR）**。它的核心思想是：**不显式建模隐藏状态 \(w_t\)，而是直接预测未来观测的概率分布。**
---

## 一、传统 POMDP 的做法

在 POMDP 中，我们假设存在一个隐藏状态 \(w_t\)：
- 转移：\(w_{t+1} \sim p(w' \mid w, a)\)
- 观测：\(o_t \sim p(o \mid w)\)
- 信念：\(b_t = p(w_t \mid h_t)\)

要预测 \(p(o_{t+1} \mid h_t, a_t)\)，我们通过信念状态：
\[
p(o_{t+1} \mid h_t, a_t) = \sum_{w_{t+1}} p(o_{t+1} \mid w_{t+1}) \sum_{w_t} p(w_{t+1} \mid w_t, a_t) b_t(w_t)
\]

这里需要：
- 隐变量 \(w_t\)
- 信念更新（贝叶斯推理）
- 转移和观测模型

---

## 二、PSR 的核心思想

**PSR（预测状态表示）** 绕开隐变量，直接建模：
\[
p(o_{t+1} \mid h_t, a_t)
\]
即：给定历史 \(h_t\) 和动作 \(a_t\)，下一个观测的概率分布。

### 关键洞察
- 如果知道**所有可能的未来观测序列**的条件概率，就足以做最优决策
- 不需要显式知道 \(w_t\)，只需要知道**预测分布**
- 这个预测分布本身就是一种“状态”——称为**预测状态**

---

## 三、PSR 的形式化

定义一组**测试（tests）**：未来观测序列 \(o_{t+1:t+k}\)。

预测状态 \(q_t\) 是这些测试的概率向量：
$q_t = [p(o_{t+1} \mid h_t, a_t), p(o_{t+1}, o_{t+2} \mid h_t, a_t, a_{t+1}), \dots]$

理论上，如果取足够多的测试，\(q_t\) 就是系统的一个**充分统计量**——和信念状态 \(b_t\) 等价，但不需要显式隐变量。

---

## 四、PSR 与 POMDP 的关系

| 维度 | POMDP | PSR |
|------|-------|-----|
| **状态表示** | 信念状态 \(b_t = p(w_t \mid h_t)\) | 预测状态 \(q_t = p(\text{future} \mid h_t, a_t)\) |
| **是否需要隐变量** | ✅ 需要 \(w_t\) | ❌ 不需要 |
| **学习目标** | 学 \(p(w' \mid w, a), p(o \mid w)\) | 直接学 \(p(o_{t+1} \mid h_t, a_t)\) |
| **更新方式** | 贝叶斯更新 | 线性更新（用 observable operators） |

---

## 五、为什么用 PSR？

| 优点 | 说明 |
|------|------|
| **无需隐变量** | 避免隐变量学习的困难（EM、变分推断） |
| **线性更新** | 在某些条件下，预测状态的更新是线性的 |
| **直接预测** | 学习目标就是预测未来观测，可监督学习 |

| 缺点 | 说明 |
|------|------|
| **测试空间巨大** | 需要选择足够多的测试来充分表示系统 |
| **可扩展性** | 在大规模问题中，测试数量可能爆炸 |

---

## 六、与 Observable Operator Models (OOM) 的关系

OOM 是 PSR 的早期形式，用**线性算子**表示系统动态：
- 状态是一个向量
- 动作对应一个矩阵（算子）
- 观测对应另一个矩阵
- 预测 = 向量 × 算子矩阵

PSR 和 OOM 共享同一个核心思想：**用可观测量的线性动态来建模系统，而不是用隐变量。**
- 提供了另一种视角：状态可以是“预测分布”，而非“隐变量分布”

一句话：**PSR 通过“预测未来观测”来定义状态，绕开隐变量，直接学习预测模型——这是对 POMDP 的一种等价但不同的数学表述。**
